# 整页 AI 图片 PPT 生产路径

## 触发

用户要求：

- 做 PPT
- 做提案
- 做方案页
- 做客户沟通页
- 做整页图 PPT
- 做图片版汇报

除非用户明确要求可编辑 PPT，否则默认走整页图路线。

## 默认链路

```text
项目经理确认边界
-> 定页序和每页一句话结论
-> 写图内文字
-> 写生图提示词和派单包
-> 生图部门生成带文字整页图
-> 用户确认关键样张
-> 扩展整套图片
-> PPT Factory 装配
-> 用户验收
```

## 图片文字硬规则

1. PPTX 不是第一产物，带文字整页图才是第一产物。
2. 图内文字是画面设计的一部分，不是后期说明层。
3. 主标题、副标题、关键词、导视和核心短句必须优先让生图模型直接生成进画面。
4. 不允许用后期叠字、PIL 写字、Canvas 文本、PPT 文本框或截图化排版补救图内文字，并把它标为正式图片页。
5. 中文错误、漏字、乱码、假字，默认重生、缩短文案或调整提示词。

## 例外

只有用户明确要求可编辑 PPT、内部结构稿或母稿时，才走可编辑路线。

例外结果必须标注：

```text
这不是生图图片成稿
```
