# 入口首轮任务定型工作流

## 触发

用户给出实质任务时触发。

纯入口句不触发主任务定型，只做恢复：

- `启动总管`
- `进入项目 <项目名>`
- `启动生图部门 <项目名>`
- `启动 PPT Factory <项目名>`

## 自动标题

纯入口句只使用恢复类临时标题；实质任务出现后，必须改为真实任务标题。

| 场景 | 标题 |
| --- | --- |
| 总管恢复 | `总管｜状态恢复` |
| 项目恢复 | `项目名｜项目恢复` |
| 生图部门恢复 | `项目名｜生图队列恢复` |
| PPT Factory 恢复 | `项目名｜PPT装配恢复` |
| 实质任务 | `项目名｜当前任务` 或 `对象｜任务` |

如果重命名工具需要 `threadId`，先读取 `CODEX_THREAD_ID`。

## 判断规则

1. 能直接做的小任务，直接执行。
2. 高风险任务先说明边界。
3. 客户可见材料先明确验收标准。
4. 图片任务转生图部门。
5. PPT 装配转 PPT Factory。
