# 使用说明书

## 日常工作流

每次进入实质任务前，总管应该先输出任务定型表和执行前置声明。入口句只做恢复，不当成主任务。

如果 Codex 支持修改线程标题，总管会按规则自动把侧栏标题改成恢复类标题或真实任务标题；如果工具要求 `threadId`，先读取 `CODEX_THREAD_ID`。

### 1. 恢复公司状态

```text
启动总管
```

总管会读取本地真源，输出任务总览板，并只推荐一个最重要下一步。

### 2. 进入项目

```text
进入项目 示例项目
```

项目经理只恢复上下文、报告状态、阻碍和下一步，不自动生产图片、PPT、PDF 或代码。

### 3. 新建任务

直接描述需求，例如：

```text
我要给一个新品牌做第一阶段增长诊断
```

总管会先做任务定型，再判断要不要进入项目、部门或工作流。

### 4. 图片任务

最终是图片时，必须走生图模型直出：

```text
启动生图部门 示例项目
```

不要让 Codex 用 HTML 截图、PIL 写字、PPT 文本框或后期叠字冒充图片成稿。

### 5. PPT 任务

默认路线：

```text
项目经理定页纲和图内文字 -> 生图部门生成带文字整页图 -> 用户确认 -> PPT Factory 装配
```

只有用户明确要求可编辑 PPT 时，才走可编辑路线。

## 常用口令

| 口令 | 作用 |
| --- | --- |
| `启动总管` | 恢复公司状态 |
| `进入项目 项目名` | 恢复项目状态 |
| `启动生图部门 项目名` | 执行图片生产 |
| `启动 PPT Factory 项目名` | 装配已确认整页图 |
| `记录` | 记录当前决策或状态 |
| `巡检` | 检查任务板、状态和边界 |

## 固定回复格式

实质任务开始时：

1. 任务定型表。
2. 执行前置声明。
3. 只推进下一步。

任务完成或发布前：

1. 完成审计表。
2. 新鲜验证证据。
3. 未覆盖风险。

## 判断标准

- 不知道下一步：先启动总管。
- 不知道项目到哪了：进入项目。
- 要做图片：生图部门。
- 要做 PPT：先确认整页图，再 PPT Factory。
- 要客户可见：必须先确认边界、素材、文案和验收标准。
