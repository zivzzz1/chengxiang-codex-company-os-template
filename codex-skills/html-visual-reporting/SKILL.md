# HTML Visual Reporting

Use this skill when the user asks for `汇报`, `复盘`, `项目简报`, `学习总结`, `可视化`, `HTML 汇报`, or wants to quickly see conclusion, evidence, risk, and next action.

Reply method:

1. Start with the professional conclusion.
2. Then show evidence, difference, risk, and next action.
3. Separate internal decision view from client preview view.
4. Do not expose internal PM/Gate/dispatch jargon in client-facing content.
5. Create an HTML file only when the user wants an artifact; otherwise use the method in the chat reply.
