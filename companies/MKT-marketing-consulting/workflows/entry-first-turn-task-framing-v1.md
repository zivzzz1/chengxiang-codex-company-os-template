# 入口首轮任务定型工作流

## 触发

用户给出实质任务时触发。

纯入口句不触发主任务定型，只做恢复：

- `启动总管`
- `进入项目 <项目名>`
- `启动生图部门 <项目名>`
- `启动 PPT Factory <项目名>`

## 自动标题

纯入口句只使用恢复类临时标题；实质任务出现后，必须改为真实任务标题。

| 场景 | 标题 |
| --- | --- |
| 总管恢复 | `总管｜状态恢复` |
| 项目恢复 | `项目名｜项目恢复` |
| 生图部门恢复 | `项目名｜生图队列恢复` |
| PPT Factory 恢复 | `项目名｜PPT装配恢复` |
| 实质任务 | `项目名｜当前任务` 或 `对象｜任务` |

如果重命名工具需要 `threadId`，先读取 `CODEX_THREAD_ID`。

## 固定回话顺序

重要任务进入实质推进前，固定按这个顺序回话：

1. 任务定型表。
2. 执行前置声明。
3. 技能路由锁定。
4. 能力触发预判面板。
5. 下一步只做。

极小单点请求可以简化，但不能漏掉用户点名的技能。

## 判断规则

1. 能直接做的小任务，直接执行。
2. 高风险任务先说明边界。
3. 客户可见材料先明确验收标准。
4. 图片任务转生图部门。
5. PPT 装配转 PPT Factory。

## 执行前置声明

进入实质推进前输出：

| 栏目 | 本轮口径 |
| --- | --- |
| 计划调用技能 / 能力 |  |
| 可能使用技能 / 能力 |  |
| 本轮不使用 |  |
| 风险闸门 |  |

不能只写“按规则执行”。必须写清楚具体哪些技能 / 能力会用、哪些不会用。

## 技能路由锁定

执行前置声明后输出：

| 栏目 | 本轮锁定 |
| --- | --- |
| 用户点名技能 | 用户原话明确点名的 Skill / 能力 / 部门；没有写无 |
| 自动命中技能 / 能力 | 由任务类型、交付物、风险 Gate 自动命中的能力 |
| 本轮明确不触发 | 容易误触发但本轮不该用的能力 |
| 完成前必须核销 | 交付前必须逐项确认的技能调用结果 |

规则：

1. 用户点名的技能必须逐项进入 `用户点名技能`，不得合并成“按要求调用技能”。
2. HTML 汇报命中 `html-visual-reporting`。
3. 一图看懂 / 信息图 / 结构图命中 `one-image-understanding`。
4. 图片最终交付命中生图模型硬 Gate。
5. 字体 / 版式 / 客户可见视觉命中字体审美检查。
6. 项目进度汇报命中项目恢复和能力触发可视审计。
7. 技能被锁定后，必须实际改变执行路径、产出结构或验收项；事后补说明不算触发。

## 能力触发预判面板

技能路由锁定后输出：

| 能力 | 是否触发 | 为什么触发 | 负责什么 | 不负责什么 |
| --- | --- | --- | --- | --- |
| <能力名> | 已触发 / 建议触发 / 不触发 | <触发原因> | <本轮职责> | <边界> |

具体审计和完成前核销见 `companies/MKT-marketing-consulting/workflows/capability-trigger-visual-audit-v1.md`。
